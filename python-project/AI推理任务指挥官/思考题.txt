1. 多线程写入 records 为什么要加 Lock？不加锁会出现什么问题？
答:records是所有线程共用的列表，多个线程同时往里面塞数据，append 不是一步完成的。
不加锁就会抢位置，出现：
有的任务记录直接消失，打印明细少几条；
数据乱套，一条任务的信息拆成两半混在别的记录里；
锁相当于排队栅栏，一次只允许一个线程写，保证每条记录完整存进去。
2. 本案例多线程能提速，纯 CPU 计算场景还能加速吗？为什么？
答:不能提速，甚至更慢。
咱们这里慢是因为time.sleep干等着，CPU 没事干，多线程可以一起等，不浪费时间，所以变快；
但 Python 有个 GIL 锁，同一时间只允许一个线程用 CPU 做计算。一堆线程疯狂算数字时，只能排队轮流上，还要来回切换线程，额外拖慢速度。
纯计算想提速只能用多进程。
3. 父类抛出 NotImplementedError 是什么设计思想，作用是什么？
答：就是定规矩：所有继承我的子类，必须自己写 predict 这个函数。
要是有人写子类忘了重写这个方法，运行代码的时候直接报错提醒，一眼就知道漏写功能了，不会跑到后面才出现莫名其妙的 bug。
